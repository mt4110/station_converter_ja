# API

The machine-readable source of truth is generated by `worker/api`.

- OpenAPI JSON: `/openapi.json`
- Docs UI: `/docs`
- Human-readable companion: [../API_SPEC.md](../API_SPEC.md)
- Design notes: [OPENAPI.md](./OPENAPI.md)

## Public Endpoints

- `GET /health`
- `GET /ready`
- `GET /v1/dataset/status`
- `GET /v1/dataset/snapshots`
- `GET /v1/dataset/changes`
- `GET /v1/stations/search`
- `GET /v1/stations/nearby`
- `GET /v1/lines/catalog`
- `GET /v1/lines/{line_name}/stations`
- `GET /v1/operators/{operator_name}/stations`

`frontend/app/api/address-search` is a Next.js helper route for the example frontend
and is intentionally outside the public `station-api` contract.

## Error Envelope

All common API errors use:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "Failed to deserialize query string",
    "detail": {
      "kind": "query_parameters",
      "issues": [{ "message": "Failed to deserialize query string" }]
    }
  }
}
```

`error.detail` is optional. Consumers can safely rely on `code` and `message`.
